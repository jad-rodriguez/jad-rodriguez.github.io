JavaScript Fundamentals

In this section, we are going to learn about some technical items, fundamental to learning basics in Javascript. This is not a comprehensive resource, but hopefully this gives you a bit of idea about it. Have fun reading!

Difference between HTML and CSS

Do you love music, or singing? Well, we'll use that analogy in helping you learn about the difference between HTML and CSS.

But first, let's define each. HTML or HyperText Markup Language is the most basic building block of the Web. It defines the meaning and structure of web content. CSS or Cascading Style Sheets describes how HTML elements are to be displayed on screen, paper, or in other media. It can also control the layout of multiple web pages all at once.

Think of HTML and CSS as the lyrics and music of a song. The lyrics creates the structure of a song, the same way HTML is the structure of a webpage. And the music which complements the lyrics, giving life to it with melodies and styles, is the same way that CSS gives life to HTML with styles.

The DOM

No, it's not "DOOM," don't worry! It's the DOM which stands for Document Object Model. The DOM or Document Object Model is a tree-like representation of the web page that gets loaded into the browser. It represents the web page using a‌‌ series of objects. The DOM is a top down representation of all the elements that make up a web page. It's the interface through which your script interacts with your HTML.

WOAH WOAH WOAH! I know that's a bit hard to digest. So let's try and give an analogy about what it is, and how we can interact with the DOM.

Let’s take a ride! Imagine a car.

If we are going to ride it, we will need...

a key.

The key serves as the bridge which allows us to interact with the car. Before we can ride the car and make it active and dynamic in the road, we need a key. In the same way, JavaScript makes makes the HTML page active and dynamic via the DOM.

But in terms of how we can think of what the DOM is and what it is made up of, we can try and imagine a house. It's got a structure, layout, sections, designs, and the list goes on and on. The DOM is made up of what's called nodes, which can be elements, attributes, text content, comments, document-related stuff, and various other things you simply never think about.

The DOM glues HTML, CSS, and JavaScript together.

Control Flow and Loops

Control Flow in JavaScript is how your computer runs code from top to bottom. It starts from the first line and ends at the last line, unless it hits any statement that changes the control flow of the program such as loops, conditionals, or functions. Think about the pipeline of water ways for example. It can run through the pipes on a specified direction, unless you create changes in the flow. In terms of water pipes, loops can be like water flowing through the pipes. The flow will be kept (for loops, kept on looping) based on the specified direction, flowing until there's nothing left to let it flow to (for loops, until there's nothing left to loop over or if the condition becomes false).

Difference between Accessing Data from Arrays and Objects

We can say that accessing data from an array is like looking at your priority list of tasks for example. You set the items in order, and you basically look at it in an ordered manner. Let’s say here are my priority of tasks today:

  1. Accomplish modules for lessons today
  2. Go for a jog
  3. Prepare dinner
  4. Do my journal

In the list above, I’ve assigned a numerical value to show the items, but even if I don’t, we basically have the idea that its order is from top to bottom, or if I put it this way [Accomplish modules for lessons today, Go for a jog, Prepare dinner, Do my journal], it’s from left to right. In Javascript, an array is an ordered list of value. Each value is called an element specified by an index. (Woah, hang on a minute, that escalated quickly!). Don’t worry, I’ll explain this to you in a moment.

In the illustration above, we can see that the items in my priority list has an assigned index (basically the equivalent numerical value in the order) of 0, 1, 2, 3. In JavaScript, the values in an array always starts with an index of 0. Simply put, we count from 0. So when you’re accessing elements (or in our example, an item in our priority list) from arrays, you are simply browsing through an ordered list of values.

How about for Objects? Well first let us try and define it first. Objects are used to represent a “thing” in your code. That could be a person, a car, a building, a book, a character in a game — basically anything that is made up or can be defined by a set of characteristics. In objects, these characteristics are called properties that consist of a key and a value.

Let's take a car for example. Car is our object, and what we put inside the curly braces ({ }), are the properties. In the illustration below you'll see that we have following properties:

  • brand: 'Subaru'
  • model: 'Impreza'
  • year: 2005
  • transmission: 'Automatic'

Properties in objects can be accessed, added, changed, and removed by using either dot or bracket notation. To get the value of the year key in our car object with both dot and bracket notation, we’d write:

  • Dot notation
    • car.year // returns 2005
  • Bracket notation
    • car['year'] // returns 2005

Functions

In JavaScript, a function allows you to define a block of code, give it a name and then execute it as many times as you want. A JavaScript function can be defined using function keyword. It allows you create instructions that will be processed to your expected results. It is compact code that is easy to understand, and are the fundamental building blocks for reusable code libraries. Here's an example of how we write this code:

//defining a function
function welcomeMessage ( ) {
alert("Hello World!");
}

A function is like a reusable set of instructions, and we'll use the voice command technology as an example to try and make it easy to understand. We are now living in a tech driven world, and I'd say that AI and "voice command" technologies are two known big things of today. It's in our devices, and now even in cars like Tesla! Imagine we can assign a voice command that will initiate set of defined instructions. I imagine saying "Schedules Today," and it will tell me all my schedules for the day with the date and time, in a specific manner I've customed it to do. I'm sure we're not too far from having a customisations like this! But enough of that for now, I really just wanna make you realise that Functions act almost like that. With functions you can create specific set of parameters, perform a specific instruction using those parameters, and will result to a reliable product.
